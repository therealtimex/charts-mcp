# Visual Testing Guide

This guide explains how to perform visual validation of charts using the example rendering scripts.

## Overview

The visual testing system generates all three output formats (`html`, `html-url`, `png`) for each example chart, allowing human reviewers to validate actual renderings.

## Quick Start

### 1. Install Dependencies

```bash
npm install
```

### 2. Start the Renderer Server

The renderer server is required for `html-url` and `png` formats:

```bash
npm run renderer:dev
```

The server will start on port 3210 (configurable via `RENDER_PORT` environment variable).

### 3. Generate Test Renders

In a separate terminal:

```bash
npm run render-examples
```

This script will:
- Read all examples from `examples/charts/` and `examples/maps/`
- Generate three outputs for each example:
  - **HTML blob** (MCP-UI format): `test-outputs/html/*.html`
  - **HTML-URL**: `test-outputs/html-url/*.txt` (contains URL)
  - **PNG**: `test-outputs/png/*.txt` (contains URL)
- Create an interactive gallery: `test-outputs/index.html`

### 4. Review the Gallery

Open the generated gallery in your browser:

```bash
open test-outputs/index.html
```

The gallery provides:
- **Summary statistics** (total examples, success rates)
- **Side-by-side comparison** of all three formats
- **Direct links** to view each format
- **PNG thumbnails** for quick visual inspection
- **File size information** for HTML blobs

## Output Formats

### 1. HTML (MCP-UI Blob)

**Location:** `test-outputs/html/*.html`

**Description:** Self-contained HTML files that can be opened directly in a browser. These represent the MCP-UI "blob" mode where HTML is embedded inline.

**Usage:**
- Click "View" in the HTML column
- Verify chart renders correctly
- Check interactivity (hover, zoom, etc.)
- Note file size (should be < 100KB for blob mode)

### 2. HTML-URL

**Location:** `test-outputs/html-url/*.txt`

**Description:** URLs pointing to interactive HTML pages hosted by the renderer server.

**Usage:**
- Click "View" in the HTML-URL column
- Verify chart renders from server
- Useful for charts > 100KB (auto-switches to server mode)

**Note:** Requires renderer server to be running.

### 3. PNG

**Location:** `test-outputs/png/*.txt`

**Description:** URLs pointing to static PNG images generated by Puppeteer.

**Usage:**
- Click "View" or check the thumbnail
- Verify static rendering quality
- Useful for reports, documentation, etc.

**Note:** Requires renderer server to be running.

## Environment Variables

### Renderer Server

```bash
# Port for renderer server (default: 3210)
export RENDER_PORT=3210

# Public base URL for generated links
export RENDER_PUBLIC_BASE=http://localhost:3210
```

### UI Resource Mode

```bash
# Control MCP-UI resource generation
# Options: "auto" (default), "blob", "server"
export UI_RESOURCE_MODE=auto

# auto: Use blob for <100KB, server for larger
# blob: Always embed HTML inline
# server: Always use external URL
```

## Visual Validation Checklist

When reviewing the gallery, check:

### ✅ Chart Rendering
- [ ] All chart types render without errors
- [ ] Data is displayed correctly
- [ ] Axes and labels are readable
- [ ] Colors match expectations
- [ ] Titles and legends appear

### ✅ Interactivity (HTML formats)
- [ ] Hover tooltips work
- [ ] Zoom/pan functionality (if applicable)
- [ ] Animations play smoothly
- [ ] Click interactions work (if applicable)

### ✅ Responsiveness
- [ ] Charts scale appropriately
- [ ] No overflow or clipping
- [ ] Mobile viewport (resize browser)

### ✅ Cross-Format Consistency
- [ ] HTML blob matches HTML-URL
- [ ] PNG captures the chart accurately
- [ ] No visual regressions between formats

### ✅ Performance
- [ ] HTML files < 100KB use blob mode
- [ ] Large charts use server mode
- [ ] PNGs load quickly
- [ ] No console errors

## Troubleshooting

### HTML-URL and PNG show errors

**Problem:** Renderer server not running

**Solution:**
```bash
# Start the renderer server
npm run renderer:dev
```

### Chart doesn't render

**Problem:** Invalid example data

**Solution:**
1. Check `examples/charts/*.json` for schema errors
2. Run schema validation: `npm test`
3. Fix data format to match chart schema

### Blank or broken charts

**Problem:** CDN library loading failure

**Solution:**
1. Check internet connection
2. Verify CDN URLs in `src/renderer/chart.ts`
3. Open browser console for errors

### File size too large

**Problem:** HTML blob > 100KB

**Solution:**
- This is expected for complex charts
- System auto-switches to server mode
- Check gallery summary for size info

## Adding New Examples

1. Create example data in `examples/charts/` or `examples/maps/`:

```json
{
  "title": "My Example Chart",
  "data": [...],
  "width": 600,
  "height": 400
}
```

2. Name file: `{chart-type}.{variant}.json`
   - Example: `bar.advanced.json`, `pie.basic.json`

3. Run validation:
```bash
npm test
```

4. Generate test renders:
```bash
npm run render-examples
```

5. Review in gallery

## CI/CD Integration

The visual testing system can be integrated into CI/CD pipelines:

```bash
# Start renderer in background
npm run renderer:dev &
RENDERER_PID=$!

# Wait for server to start
sleep 2

# Generate renders
npm run render-examples

# Upload artifacts
# (upload test-outputs/ to artifact storage)

# Cleanup
kill $RENDERER_PID
```

## Tips

1. **Use the gallery** - Don't manually check each file; the gallery provides efficient side-by-side comparison

2. **Check sizes** - HTML blobs > 100KB should trigger server mode automatically

3. **Test offline** - HTML blobs should work offline; URLs require server

4. **Screenshot regressions** - Save PNG URLs for baseline comparisons

5. **Mobile testing** - Resize browser to check responsive behavior

6. **Theme testing** - Add examples with different themes (dark, academy, etc.)

## FAQ

**Q: Can I run this without the renderer server?**

A: Yes, but only HTML blob format will work. HTML-URL and PNG require the server.

**Q: How do I test a specific chart type?**

A: Modify `scripts/render-examples.ts` to filter by chart type, or manually check `test-outputs/index.html` and filter the table.

**Q: Can I use this for regression testing?**

A: Yes! Save PNG URLs as baselines and compare future renders using image diff tools.

**Q: What if a chart fails to render?**

A: Check:
1. Example data schema validity (`npm test`)
2. Browser console errors (open HTML in browser)
3. Renderer server logs
4. Chart implementation in `src/charts/`
